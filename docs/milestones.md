# Milestones

## Milestone A — Fork boots, unchanged
**Deliverable:** Vimtype runs locally and via Docker with stock Monkeytype behavior.

**Exit criteria:** baseline app works before modifications.

## Milestone B — Algorithms mode without Vim
**Deliverable:** Algorithms content is rendered and scored via existing pipeline.

**Exit criteria:** complete an algorithm test and see a normal results screen.

## Milestone C — Vim Insert Mode only
**Deliverable:** Insert-only Vim editor is playable end-to-end.

**Exit criteria:** results match Monkeytype scoring for insert-only flow.

## Milestone D — Normal mode + core motions
**Deliverable:** Normal/Insert transitions and core motions/editing are usable.

**Exit criteria:** users can navigate and correct mistakes in Vim style.

## Milestone E — Visual mode
**Deliverable:** Visual char/line selection with delete.

**Exit criteria:** visual selection works and tests remain completable.

## Milestone F — Scoring correctness parity
**Deliverable:** Scoring logic matches Monkeytype with Vim navigation/editing.

**Exit criteria:** results are stable, comparable, and not trivially gameable.

## Milestone G — Backend compatibility + leaderboards
**Deliverable:** backend accepts results and leaderboards behave consistently.

**Exit criteria:** server accepts submissions for algorithms/vim if enabled.

## Milestone H — Production hardening
**Deliverable:** deployable, monitorable, maintainable setup.

**Exit criteria:** monitoring, analytics, security controls, and CI are in place.
